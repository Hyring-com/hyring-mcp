# Skill: Two-Way (Dynamic) AI Interview

**Assessment Type:** `dynamic`
**Backend Port:** 5000 (screener API)
**Frontend:** `ai-screener-frontend`

---

## What It Is

An adaptive, conversational AI interview. The AI (not a human) conducts a real-time two-way conversation with the candidate. Questions are generated dynamically based on the candidate's responses and the skill context set by the employer. There are no pre-written fixed questions.

---

## Creation Flow (3 Steps)

```
Step 1: POST /employer/assessment/job-interview/{employerId}
  Body: { assessmentType: "dynamic", avatarType, aiVoice, language }
  Response: { assessmentId (UUID), employerId }

Step 2: POST /employer/assessment/job-details/{employerId}
  Body: { assessmentId, jdText }
  Response: { status: "JD_UPDATED" }

Step 3: PATCH /employer/assessment/job-details/{employerId}
  Body: {
    assessmentId,
    roleName, seniorityLevel, employmentType,
    workPlaceType, jobLocationCountry, jobLocationCity,
    skills, currency, annualSalaryRangeFrom, annualSalaryRangeTo
  }
  Response: { status: "JOB_DETAILS_UPDATED" }
```

**MCP Tool:** `create_dynamic_assessment` ✅

---

## Skill Context Setup

Instead of fixed questions, the employer provides skill areas for the AI to probe.

**MCP Tool:** `set_interview_context`
**Endpoint:** POST `/employer/assessment/create-context/{employerId}`

```json
{
  "assessmentId": "uuid",
  "skills": [
    {
      "skill": "React",
      "difficulty": "Hard",
      "concepts": ["hooks", "context API", "performance optimization"]
    },
    {
      "skill": "Node.js",
      "difficulty": "Moderate",
      "concepts": ["event loop", "streams", "REST APIs"]
    }
  ]
}
```

**Constraints:**
- Minimum 3 skills, maximum 5 skills
- Difficulty: `Easy` | `Moderate` | `Hard`

## Interview Duration

**Fixed:** 10 minutes (600 seconds) regardless of skills set.

### Frontend Calculations on This Data

1. **Technical score (per skill):**
   ```
   For each skill conversation:
     score = isOverwritten ? overWrittenScore : score
   Average all skill scores → technical_score (0-10)
   Normalize → 0-100 scale
   ```

2. **Communication score (from language_score):**
   - Pronunciation: `(pronunciation_score * 10) / 100`
   - Grammar: `(grammar_score * 10) / 100`
   - Vocabulary: `(vocabulary_score * 10) / 100`
   - Filler Words: `(filler_score * 10) / 100`
   - Fluency: `(fluency_score * 10) / 100`
   - Average of all 5 → communication score (0-100)

3. **Fit score:**
   ```
   fit_score = tech_score * (weightage.technicalScore / 100)
             + comm_score * (weightage.communicationScore / 100)
   ```
   Default weights: Technical 50%, Communication 50%

4. **Fit level label:**
   - 0-25 → WEAK FIT
   - 26-50 → MODERATE FIT
   - 51-75 → GOOD FIT
   - 76-100 → STRONG FIT

5. **General score labels:**
   - ≤30 → POOR
   - ≤50 → BELOW AVG.
   - ≤70 → AVERAGE
   - ≤90 → GOOD
   - >90 → EXCELLENT

### MCP Tool
**`get_dynamic_report`** - Fetches data and applies above calculations.

---

## What the MCP Report Shows

- Candidate name, email, completion status
- Fit score (0-100) with label
- Technical score (0-100) with label - averaged across all skills
- Communication score (0-100) with label
- Per-skill breakdown:
  - Skill name and difficulty
  - Score (0-10) with label
  - Conversation transcript (AI questions + candidate answers)
- Communication sub-scores:
  - Pronunciation, Grammar, Vocabulary, Filler Words, Fluency (each 0-100)
- Accent analysis
- Video link
- Screen share link (if enabled)
- AI-generated summary
- Proctoring data

---

## Key Difference from One-Way

| Aspect | One-Way (Fixed) | Two-Way (Dynamic) |
|--------|-----------------|-------------------|
| Questions | Pre-written by employer | Generated by AI in real-time |
| Interaction | Candidate answers alone | AI converses with candidate |
| Transcript | Per-question | Per-skill conversation |
| Screen share | No | Yes |
| Duration | Sum of question times | Fixed 10 min |
| Skill depth | Question-level | Skill-conversation level |
